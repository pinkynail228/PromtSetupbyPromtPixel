# Serena-First Workflow

Serena is the default context system. Use it before manual repository exploration.

## Mandatory protocol

1. Do not start with broad manual file exploration.
2. Start with Serena project activation.
3. Confirm onboarding status.
4. Load relevant memories for the task.
5. Only then run targeted manual search if required.

## Before task

1. Activate project in Serena.
2. Check onboarding status.
3. Load relevant memories before planning.

## During task

1. Record key decisions and conventions in Serena memory.
2. Keep memory entries factual and concise.
3. Write memory updates when introducing durable patterns.

## After task

1. Write summary memory: changes, checks, risks.
2. Note pitfalls and follow-up recommendations.
3. Ensure next contributor can continue from memory without re-reading large code areas.
